// What Cadence does

Everything email. Nothing extraneous.

01

Simple sending API

REST API and SDKs for 11 languages. Call email.send() with a template name and data object. That's it for the common case.

02

Deliverability tooling

SPF, DKIM, DMARC with guided DNS setup. Suppression lists maintained automatically. Reputation monitoring with proactive alerts. No deliverability consultant needed.

03

React component templates

Write email in JSX. Render to battle-tested HTML. Version-control your templates in git. Engineers own the structure; content teams own the words.

04

Preview & test mode

Render any template with real or mock data in the browser before a single production send. Test mode captures sends without delivering them — perfect for staging environments.

05

Analytics per message

Opens, clicks, deliveries, bounces, and unsubscribes tracked at the message level and rolled up by template. Exportable via webhook or the Events API.

06

Dashboard for non-engineers

Content editors update copy, preview results, and schedule campaigns without a deploy. Role-based access means they never touch API credentials.

// Deliverability

The hard, invisible part of email. Handled.

Getting email into the inbox — rather than spam — requires constant, un-glamorous infrastructure work. Most teams discover this the hard way when their welcome email starts hitting promotions folders. Cadence ships with everything required, configured correctly, from the first send.

Read the deliverability guide →
Guided SPF, DKIM, and DMARC setup

Step-by-step DNS instructions per domain. We verify records after you add them and monitor continuously — if a record drifts or gets deleted, we alert you within minutes.

Automatic bounce and complaint handling

Hard bounces and spam complaints come back via feedback loop. Cadence suppresses the address immediately and removes it from future sends — no manual intervention required.

Managed IP warm-up

New accounts start on shared sending pools with established reputation. As volume grows, we move senders to dedicated IPs with a managed warm-up schedule.

Reputation dashboard

Real-time view of your sender reputation, complaint rates, and bounce rates. Thresholds alert you before ISPs start filtering your mail.

Smart throttling and retry logic

Cadence detects temporary delivery failures (soft bounces, rate limits from receiving servers) and retries with correct backoff — without requiring you to implement it.

// One API, eleven SDKs

Works in the language you're already using.

Ruby
client = Cadence::Client.new(api_key: ENV['CADENCE_KEY'])

client.email.send(
  to:       'user@example.com',
  from:     'app@yourco.com',
  template: 'subscription-renewed',
  data:     { plan: 'pro', amount: 49 }
)
Python
from cadence import Cadence

client = Cadence(api_key=os.environ["CADENCE_KEY"])

client.email.send(
    to="user@example.com",
    from_="app@yourco.com",
    template="invoice-ready",
    data={"invoice_url": invoice_link},
)
Go
c := cadence.New(os.Getenv("CADENCE_KEY"))

_, err := c.Email.Send(ctx, cadence.SendParams{
    To:       "user@example.com",
    From:     "app@yourco.com",
    Template: "plan-downgraded",
    Data:     map[string]any{"plan": "free"},
})
cURL
curl -X POST https://api.cadencemail.com/v1/email/send \
  -H "Authorization: Bearer $CADENCE_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "to":       "user@example.com",
    "from":     "app@yourco.com",
    "template": "password-reset",
    "data":     { "reset_url": "https://..." }
  }'
