One decorator. Infinite scale.

Write ordinary Python. Voltage handles everything between your local call and the cloud result.

01

Decorate your function

Add @voltage.function and declare what you need — GPU model, memory, timeout. That's it. Voltage builds the environment from your existing requirements.txt.

02

Call it with .remote()

Swap fn(args) for fn.remote(args). Voltage cold-starts a container, runs your code, streams logs back. No CLI pipelines to configure.

03

Scale — automatically

Call it 1000 times in a loop and Voltage fans out to 1000 parallel containers. When they finish, they scale back to zero. You pay only for active compute seconds.

Parallel execution

Fan out to thousands. Collect results as futures.

Use voltage.map() or scatter calls across a list. Every invocation is independent, scheduled on the best available hardware, and results are gathered back asynchronously.

  • ▸ Fan out 10,000 inference calls in a single line
  • ▸ Gather partial results as they complete
  • ▸ Automatic retry on transient failures
  • ▸ Mix GPU and CPU functions in the same call graph

inference_api.py 
@voltage.endpoint(
    gpu="A10G",
    min_containers=1,   # keep warm
    max_containers=50,  # burst capacity
)
class InferenceAPI:
    def __init__(self):
        self.model = load_model("./weights")

    @voltage.web_endpoint(method="POST")
    def predict(self, prompt: str) -> dict:
        return {"output": self.model.generate(prompt)}

# Deploy: voltage deploy inference_api.py
# → https://api.voltage.run/your-app/predict
