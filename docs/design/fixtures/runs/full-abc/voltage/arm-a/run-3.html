Capabilities
GPU compute

The right GPU, attached instantly.

A100, H100, L40S — specify in a decorator argument. Voltage provisions the instance, attaches the GPU, and runs your function. It scales down when finished. No reservations required.

@voltage.function(gpu="H100", memory="80GB")
Parallel scale

Map over thousands. Collect results.

Pass a list of inputs to .map(). Voltage distributes work across parallel containers, returns results in order. No queue configuration, no retry logic to write yourself.

results = process.map(shards, concurrency=2000)
Inference endpoints

A URL for any model, instantly.

One decorator turns a function into a persistent HTTP endpoint. Auto-scales to traffic, returns to zero when idle. No load balancers. No Kubernetes. No ops overhead.

@voltage.endpoint(min_replicas=0)
Scheduling

Cron jobs in Python, not YAML.

Schedule nightly retraining, data pipelines, model evaluations. Express schedules as Python — trigger from cron, CI, webhooks, or other functions. Monitor from the Voltage dashboard.

@voltage.cron(schedule="0 2 * * *")
The premise

Infrastructure shouldn't be the hard part.

ML engineers spend weeks writing Dockerfiles, YAML manifests, and Kubernetes configs — before a single training run. Voltage makes this work disappear.

Write ordinary Python. The decorator is the infrastructure. Voltage handles provisioning, networking, scaling, and teardown — automatically, reliably, cheaply.

No Docker

Voltage builds container images from your Python environment automatically. Specify dependencies in a decorator argument, not a Dockerfile.

No YAML

Resource requirements, scaling limits, and schedules are Python function arguments. They live next to your code.

No cluster ops

No GPU nodes to manage, no autoscaling policies to tune, no on-call rotation for your compute infrastructure.
