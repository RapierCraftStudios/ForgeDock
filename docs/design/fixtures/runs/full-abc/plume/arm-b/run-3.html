Features

Designed around how
engineering teams actually work.

Every feature in Plume exists to save time or eliminate a meeting. Nothing else made the cut.

Keyboard-first navigation

Every action is reachable from the keyboard. Create, assign, prioritize, move to cycle — without leaving the home row.

Cycles without the ceremony

Time-boxed sprints that auto-carry incomplete work. Start a new cycle in one click. Analytics roll up automatically.

Live roadmap for leads

Quarter and year views with real-time progress from issues. No manual status updates. No deck to maintain the night before planning.

GitHub, GitLab, Bitbucket

Branch names auto-link. PRs surface in issue context. Status changes on open, review, merge, and CI pass or fail.

Projects with milestones

Group ongoing work into projects. Milestones track delivery dates. Issues belong to a project and a cycle at the same time.

Command palette

Hit ⌘K from anywhere. Search issues, jump to projects, run bulk actions, navigate views. Two keystrokes to anything.

Cycles

Sprint planning without the planning meeting.

Cycles are time-boxed containers for issues. Start one with a click, drag in what matters, and Plume tracks progress automatically. When the cycle ends, incomplete work carries over — nothing falls through the cracks.

Common questions

How is Plume different from Linear or Jira?

Plume is more opinionated than Linear and far leaner than Jira. We made deliberate choices about what to leave out. There are no points, no boards unless you want them, no 12-step issue creation flows. If you want to add a ticket in under 3 seconds, Plume was built for that.

Can I import from Jira or GitHub Issues?

Yes. We support one-click import from Jira, GitHub Issues, Linear, Asana, and CSV. Imports preserve titles, descriptions, assignees, labels, and statuses. Most teams are fully migrated in under 30 minutes.

Do product and design people use it too?

Yes — and this is intentional. Plume is built so engineering, product, and design share one source of truth. PMs manage the backlog and roadmap; designers track work in the same issues as engineering. No translation layer needed.

Is there an API?

Yes. Plume has a full REST API and webhooks. You can create issues, update statuses, query cycles, and build integrations. SDKs for TypeScript and Python are available. Docs live at api.plume.dev.

What happens if my team grows past 10?

You get a 14-day heads-up and can upgrade to Pro at any time. There's no data loss, no migration, no interruption. You pick a billing date that works for your team.
