Everything you need.
Nothing you don't.

Plume is opinionated about workflow. Every feature exists because shipping software demands it. No feature exists because a sales deck required it.

01
Issues
The atomic unit of work. Status, priority, assignee, labels, rich markdown. Create in under a second. Update inline. No modals, no save buttons, no page reloads.
Sub-issues. Bulk operations. Custom fields. Image paste. Syntax highlighting in descriptions. Filter by any combination.
02
Cycles
Time-boxed sprints that keep your team in rhythm. Drag issues in, track progress, review what shipped. Unfinished work rolls forward. No ceremony.
1-4 week duration. Burndown charts. Velocity tracking. Scope-change alerts. Auto-archive on close.
03
Projects
Long-running initiatives that span cycles. Milestones, progress tracking, roadmap views. See which sprints contributed and how much is left.
Timeline view. Rich status updates. Automatic progress from issues. Project-scoped boards and filters.
04
Views
Board, list, timeline. Filter by status, priority, assignee, label, cycle, project. Save views for yourself or your team. Every view loads instantly.
Custom saved views. Shareable URLs. Group by any field. Sort by priority, date, or custom order.
05
Speed
Engineered for sub-50ms interactions. Optimistic UI, local-first data, incremental sync. No spinners. No loading states. Your attention is never interrupted.
Offline-capable. Background sync. Instant navigation. Sub-second page loads on any connection.
06
API
Everything in Plume is accessible via REST and GraphQL. Build custom workflows, automate triage, integrate with internal tools. First-class developer experience.
Webhooks. OAuth2 apps. Rate limiting with clear headers. Comprehensive docs with runnable examples.

Capture. Plan.
Build. Ship.

Four phases. One tool. No tab-switching, no data entry, no status meetings about statuses.

01
Capture
Press C. Type the title. Set priority. Done. Issues get a unique ID, a place in the backlog, and context from the start. No forms, no mandatory fields.
Also: create from Slack, from GitHub comments, from the API. Templates for bugs, features, tasks.
→
02
Plan
Drag issues into cycles. Set scope. Track velocity from past sprints to know what fits. Scope-change tracking protects your commitments mid-sprint.
Backlog grooming. Effort estimates. Dependency flags. Capacity calculation from team size and history.
→
03
Build
Issues move as code ships. PRs link by branch name. Statuses transition on merge. The board reflects reality without anyone touching it.
GitHub/GitLab sync. Branch name patterns: plm-163-fix-tz. Review status on the issue. CI status inline.
→
04
Ship
Close the cycle. Review velocity. Archive what shipped. Unfinished work rolls forward. Your team knows exactly what landed and what's next.
Cycle summaries. Changelog generation. Slack notifications. Historical velocity trends for better estimates.

your stack.

Plume connects to the tools your team uses. No data silos, no context switching, no copy-paste between tabs.

  • GH

    PRs, branches, commits. Auto-link by branch name. Status transitions on merge. Bi-directional sync.
  • GL

  • SK
    Slack
    Create issues from messages. Status change notifications. /plume slash commands for quick actions.
  • FG

    Embed designs in issues. Inline thumbnails. Keep design and engineering context together.

    Auto-create issues from production errors. Stack traces linked. Close on deploy.
