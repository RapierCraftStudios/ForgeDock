Designed around how
teams actually work

Every decision in Plume serves one goal: remove friction between having an idea and shipping it.

01

Issues that move

Create, triage, and close issues faster than you can context-switch. Status, priority, labels, and assignees update in place. No page reloads, no save buttons, no waiting.

Custom statuses per team. Bulk operations on any selection. Sub-issues for decomposition. Markdown descriptions with inline code and image paste.
02

Cycles keep cadence

Time-boxed sprints that start automatically. Drag issues in, track velocity, review what shipped. Unfinished work rolls forward with a single action. No ceremony.

Configurable cycle length (1-4 weeks). Burndown and velocity charts. Auto-archive completed cycles. Scope-change tracking to protect commitments.
03

Projects span cycles

Long-running initiatives that cross multiple sprints. Set milestones, track progress as a percentage, and see which cycles contribute. Projects give you the big picture without a separate tool.

Roadmap timeline view. Status updates with rich text. Automatic progress from child issues. Filter the board by project to focus.

From idea to shipped
in one place

Plume replaces the constellation of tools your team has outgrown. One workspace, one source of truth, no tab-switching.

01
Capture
Create issues in seconds. Keyboard shortcut, command palette, or Slack integration. Every issue gets a unique ID and immediate context.
Quick-capture with C. Paste images inline. Templates for bugs, features, and tasks. Issues link to code from the start.
02
Plan
Drag issues into cycles. Set priorities. Scope work that fits your team's velocity. Protect the sprint by tracking scope changes in real time.
Backlog grooming view. Effort estimates. Dependency flags. Automatic capacity calculation based on team size and historical data.
03
Build
Issues move through statuses as code ships. PRs link automatically. Branch names generate from issue IDs. The board updates without anyone touching it.
GitHub and GitLab sync. Auto-transition on PR merge. Branch name patterns: plm-163-fix-timezone. Code review status reflected on the issue.
04
Ship
Close the cycle, review velocity, archive what shipped. Unfinished work rolls forward. The team knows exactly what landed and what's next.
Cycle summary reports. Changelog generation. Slack notifications on cycle close. Historical velocity trend so estimates improve over time.

existing stack
