From zero to inbox in four steps

No SMTP config. No IP warm-up guesswork. No deploy-to-change-copy cycles.

1

Create your account & get an API key

Sign up, verify your sending domain, and copy your cd_live_ key. Domain verification takes about 2 minutes if you have DNS access.

2

Install the SDK

npm install cadence or pip install cadence. SDKs also available for Ruby, Go, PHP, and Java. Or call the REST API directly.

3

Create a template or send raw HTML

Use the template editor to design your email, or pass raw HTML in the API call. Templates accept variables like {{name}} and support conditional blocks.

4

Call cadence.send()

Pass the recipient, template slug, and data. Cadence handles rendering, delivery, bounce processing, and analytics. You get a message ID back instantly.

What you get

A complete email platform behind a clean API surface.

API

RESTful sending API

Single endpoint to send email. Supports batch sends (up to 1,000 recipients per call), scheduled delivery, and idempotency keys for safe retries.

Deliverability

Inbox, not spam folder

Automated domain authentication (SPF, DKIM, DMARC). Dedicated IP pools. ISP feedback loops. Bounce and complaint auto-suppression. We monitor sender reputation so you don't have to.

Templates

React-style email components

Build emails from composable components: headers, footers, product cards, CTAs. Or use the drag-and-drop editor. Either way, non-engineers can edit content without a deploy.

Analytics

Know what happens after send

Per-message and aggregate stats for deliveries, opens, clicks, bounces, and unsubscribes. Real-time webhooks for every event. Export to S3, BigQuery, or Snowflake.

Testing

Preview before you ship

Test mode captures every email in a virtual inbox. Preview rendered templates with live data. Integration tests can assert on email content without sending real mail.

Collaboration

Engineers + content owners

Role-based access: developers manage API keys, domains, and webhooks. Content owners manage templates, schedules, and campaigns. Shared analytics dashboard for both.
