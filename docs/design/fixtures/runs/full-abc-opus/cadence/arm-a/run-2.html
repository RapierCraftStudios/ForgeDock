SDKs for five languages. One consistent API.

Every SDK wraps the same REST API. Same method names, same error handling, same response shapes. Pick the one that matches your stack.

const result = await cadence.send({

What you get.

Every feature below is available on every plan. Usage-based pricing, no feature gates.

Capability Description API
Transactional send Single-recipient email with template or inline HTML. Supports attachments, inline images, custom headers, and reply-to. POST /v1/emails/send
Batch send Up to 10,000 recipients per call with per-recipient data merge. Scheduled or immediate. Progress tracked via webhook. POST /v1/emails/batch
Template management Version-controlled templates with variables, conditionals, loops, and component includes. Visual editor for non-engineers. GET/POST /v1/templates
Domain authentication Automated SPF, DKIM, DMARC, and BIMI setup. Continuous monitoring with alerts on DNS drift or key expiry. POST /v1/domains/verify
Suppression lists Global and per-account suppression. Auto-add on hard bounce, complaint, or unsubscribe. Manual import/export. GET/POST /v1/suppressions
Webhook events Real-time delivery, open, click, bounce, complaint, and unsubscribe events. Retry with exponential backoff. POST /v1/webhooks
Analytics Aggregate and per-message analytics. Filter by template, tag, domain, or date range. CSV and API export. GET /v1/analytics
Test mode Sandbox sending that captures email without delivering. Full header inspection and rendering preview. X-Cadence-Test: true

Get Your API Key Read the Docs
01 Sign up and grab your API key from the dashboard. No credit card required.
02 Verify your domain by adding the DNS records Cadence generates. Takes about 2 minutes.
03 Install the SDK: npm i @cadence/node (or pip, go get, gem install)
04 Send a test email with X-Cadence-Test: true header. Inspect it in the dashboard.
05 Go live. Remove the test header and start sending. Monitor delivery in real time.
