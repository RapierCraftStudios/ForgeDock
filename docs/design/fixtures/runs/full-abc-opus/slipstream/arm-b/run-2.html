What the platform does

Previews

Every PR is a running environment

When a developer opens a pull request, Slipstream builds and deploys it to a unique URL. Reviewers see the change running, not just as code. Comments and approvals happen alongside the live preview.

Edge Network

Fast everywhere, not just near your servers

Static assets are cached at 94 edge locations. Dynamic routes execute at the edge closest to the user. No regions to choose, no replicas to configure. Your app is fast by default.

Serverless Functions

Backend logic, zero server management

Write API routes and server-side logic as functions. They deploy with your frontend, scale automatically, and run in the same edge network. You never provision a server or choose an instance type.

// api/hello.ts
export default function handler(req, res) {
  const user = await db.users.find(req.query.id)
  return res.json({ name: user.name, plan: user.plan })
}
// Deployed to the edge. Scales to zero. No servers.
Git Integration

Your repository is the source of truth

Connect GitHub, GitLab, or Bitbucket. Every push to any branch triggers a build. Merge to main and it's in production. Rollback is a revert commit. The workflow is Git, nothing else.

Collaboration

Ship together, not just side by side

Deploy notifications in Slack. Comments on preview deployments. Branch-level permissions and approval gates. Your whole team sees what shipped, who shipped it, and when.
