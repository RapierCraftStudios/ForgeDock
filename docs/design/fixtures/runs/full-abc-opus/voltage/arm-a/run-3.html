How it works

Three steps. No infrastructure.

01 / Define

Write a Python function

Use the @app.function decorator to specify GPU type, memory, timeout, and dependencies. Your function is the deployment artifact.

@app.function(gpu="A100")
02 / Deploy

One command to ship

Voltage builds a container with your dependencies, pushes it to our global fleet, and creates an HTTPS endpoint automatically.

$ voltage deploy
03 / Scale

Automatic scaling

From zero to thousands of GPU containers based on demand. Scale back to zero between bursts. Per-second billing only.

generate.remote(prompt)

Patterns

What people build

Batch processing

Parallel GPU inference at scale

Fan out across hundreds of GPU containers with .map(). Process millions of items in minutes instead of hours.

@app.function(gpu="T4", concurrency=200) def embed(batch: list[str]): model = SentenceTransformer("all-MiniLM-L6-v2") return model.encode(batch) # 5M docs across 200 containers results = list(embed.map(chunks))
Scheduled jobs

Nightly training with GPU on demand

Schedule training runs or batch jobs. GPU allocated only during execution, released immediately after.

@app.function( gpu="A100-80GB", schedule=voltage.Cron("0 2 * * *"), timeout=7200, ) def nightly_train(): dataset = load_latest_data() model = finetune(dataset, epochs=5) push_to_registry(model)
Web endpoints

Model serving with auto-scaling

Turn any model into an HTTP API with automatic scaling, request batching, and GPU warm pools.

@app.endpoint(method="POST") def generate(prompt: str): return TextToImage().generate.remote( prompt ) # https://your-app.voltage.run/generate
Data pipelines

Chain functions into workflows

Build DAG pipelines that mix CPU and GPU stages. Each stage scales independently. Shared volumes pass data between steps.

@app.function(cpu=8) def preprocess(raw_path: str): data = clean(load(raw_path)) return save(data, "/vol/clean") @app.function(gpu="A100") def train(data_path: str): model = fit(load(data_path)) return save(model, "/vol/models")
