import voltage
from voltage import Volume

# Persistent storage for model checkpoints
checkpoints = Volume.persisted("training-checkpoints")

@voltage.function(
    gpu="H100",
    memory=65536,
    volumes={"/checkpoints": checkpoints},
    timeout=3600,
)
def train_lora(base_model: str, dataset_path: str):
    from transformers import AutoModelForCausalLM
    from peft import LoraConfig, get_peft_model

    model = AutoModelForCausalLM.from_pretrained(
        base_model, device_map="auto"
    )
    peft_model = get_peft_model(model, LoraConfig(
        r=16, lora_alpha=32, target_modules=["q_proj", "v_proj"]
    ))

    # ... training loop ...
    peft_model.save_pretrained("/checkpoints/lora-adapter")

Platform capabilities
Autoscaling

Zero to 10,000 containers in seconds

Voltage scales your function horizontally based on request volume. When traffic drops, containers are reclaimed instantly. You never pay for idle hardware. Min and max concurrency are configurable per function.

GPU Fleet

A100, H100, L4, T4

Select GPU type per function. Multi-GPU configurations available for large model training. Spot pricing for fault-tolerant batch workloads.

Endpoints

Serve models over HTTPS

Turn any function into an inference API with one flag. Built-in request queuing, batching, and auth. WebSocket streaming for generative models.

Scheduling

Cron and event-driven triggers

Schedule recurring jobs or trigger functions from webhooks, queues, or other Voltage functions. Build entire ML pipelines as composable functions.

Developer Experience

A local-feeling development loop

Run functions locally with voltage run --local, then deploy with voltage deploy. Hot-reload during development. Tail logs, stream metrics, and SSH into running containers for debugging. The CLI is designed to stay out of your way.
