# Brief — Slipstream

> Design-blind seed brief. Describes only what the product does. No visual direction.
> Anonymized name — do not infer or reproduce any real brand.

## Product

**Slipstream** is a platform for building and deploying web applications. Developers connect a git
repository and every push is built and deployed automatically to a global edge network, with preview
environments for each change and zero server management.

## Who it's for

Frontend and full-stack developers and the teams shipping web apps who want to deploy without operating
infrastructure.

## What it does

- Git-connected deploys: push to a branch, get a live URL.
- Preview deployments for every pull request, so changes can be reviewed running, not just as a diff.
- A global edge network and CDN so apps are fast everywhere by default.
- Serverless and edge functions for backend logic without managing servers.
- Collaboration features so a whole team can see and comment on deploys.

## What the landing page must communicate

- Speed and immediacy: from push to production is effortless and fast.
- Zero infrastructure burden — no servers, no ops.
- It is the obvious home for modern frontend frameworks.
- Confidence and restraint: this is a serious, opinionated platform.

## Primary call to action

Deploy now / start for free.

## Constraints

- Concrete and direct copy; let the capability speak.
- Reads as a stark, confident, well-funded developer platform.
