## Canonical Description

Use this across all submissions (≤150 chars):

> **ForgeDock** — Autonomous AI dev pipeline for Claude Code. Uses GitHub issues/PRs as a structured knowledge graph so agents never lose context between sessions.

Long form (for platforms that allow more):

> ForgeDock turns GitHub into a persistent knowledge graph for Claude Code agents. Every pipeline stage (investigate → architect → build → review → merge) writes structured annotations to GitHub issues and PRs — so agents always know what happened before them, why the code looks the way it does, and what to do next. Install with `npx forgedock` and run `/work-on #42`.

## Submission Guidelines (General)

1. **Follow each repo's CONTRIBUTING.md** — most awesome lists have specific format requirements
2. **Alphabetical order** — most lists are alphabetically sorted; find the correct position
3. **One PR per list** — do not batch submissions
4. **Description length** — most lists cap at 120–150 chars; use the canonical short description
5. **No self-promotional language** — "the best", "revolutionary", "amazing" are red flags
6. **Include the category** — verify the correct section before submitting
7. **Check for duplicates** — search the list for "forgedock", "claude-code", "forge" before PRing
